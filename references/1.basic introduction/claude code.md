# 介绍

**Claude Code** 是由 Anthropic 推出的一个 **AI 编程 Agent（智能编程代理）**，它并不是一个聊天机器人，也不是传统意义上的代码补全工具，而是一个能够**理解整个项目、主动执行开发任务**的 AI 开发助手

它主要运行在==终端（Terminal）==中，也可以与 IDE、GitHub 等工具协同工具

**注**：区分 claude 和 claude code ； claude 是模型名称，claude code 是 Agent 工具

---

# 版本（claude code）

##  **1. CLI 版（Terminal / 命令行版）** —— 功能最完整的核心版本

- 这是 Claude Code 的**原生形态**，功能最全
- 直接在终端中运行，支持完整本地环境、脚本、CI/CD 等高级用法
- 适合重度开发者

==详细介绍在下方，后续我们使用的就是这个版本==

## **2. Desktop 版（桌面应用）** —— 图形化 + 功能均衡的最佳平衡点

- 一款独立的 **原生桌面应用**，把 Chat、Code、Cowork 等功能整合在一起
- **同时支持两种会话模式**：
    - 本地会话：直接操作你电脑上的文件系统（和 CLI 类似功能）
    - 云端会话：类似 Web 版，在 Anthropic 云端运行
- **图形化体验**（最大亮点）：
    - 视觉 Diff 查看器（并排对比代码变更）
    - 应用实时预览（运行 Web/移动应用时可直接看到效果）
    - 并行会话管理界面（轻松切换多个任务）
    - 更好的进度可视化、变更总结、评论功能
- **其他优势**：
    - 支持 **Dispatch** 功能（手机上启动任务，推送到 Desktop 执行）
    - 支持 **Computer Use**（在 Pro/Max 计划下可操作电脑界面）
    - 配置更友好（图形界面设置 MCP、Skills 等）
    - 稳定性高，适合长时间使用
**缺点**：
- 功能完整度略低于 CLI（部分高级脚本、Agent SDK、某些 Hooks 仍以 CLI 更强）
- Shell 可能有限制（部分系统默认使用 bash）

==其实这个版本非常好用，但桌面版对账号的要求及其的高，中国内地用户很难使用==

## **3. IDE 集成版** —— 与编辑器深度融合

这是很多开发者**日常使用频率最高**的形态
 **VS Code 扩展**（含 Cursor 等 VS Code Fork）
- **最受欢迎**的集成方式
- **核心优势**：
    - **内联 Diff**：Claude 修改代码后，可以直接在编辑器中以高亮方式显示变更，一键接受/拒绝
    - 侧边栏聊天窗口，与当前打开的文件自动关联上下文
    - 支持文件 @ 提及、选中代码快速提问
    - 集成终端，可在 VS Code 内直接运行 Claude Code
    - Checkpoint（检查点）功能：可回滚到之前的会话状态
- 几乎感觉不到在用“另一个工具”，无缝嵌入工作流

**注**：VS code 是微软开发的免费开源代码编辑器，轻量强大、插件生态丰富，支持几乎所有主流编程语言

==IDE 的场景为程序开发，所以不符合我们的使用场景，暂时可以不考虑==

# 对比

## 1.  网页版 AI VS claude code
Claude Code 与网页版 AI 的本质区别在于：一个是“思考者”（聊天助手），另一个是“行动者”（自主编码代理）

**网页版 AI**：
- 通用对话式 AI 助手
- 擅长思考、生成、分析：写文章、头脑风暴、总结文档、回答问题、生成代码片段、分析数据等
- 你需要手动提供上下文（复制粘贴代码、上传文件），它基于对话给出建议或输出
- 它是被动的：**你问它答，主要输出文本**

**Claude Code**：
- **Agentic**，专为开发者设计，能**自主行动**
- 它可以**直接操作电脑文件**：读取整个项目文件、编辑多文件、运行命令/测试、修复 bug、提交 Git、创建 PR 等
- 它是**主动的**：给你一个任务，它会自己规划、执行、验证、迭代，直到完成（类似一个会编程的同事）

| 维度       | 网页版 Claude (claude.ai) | Claude Code                   |
| -------- | ---------------------- | ----------------------------- |
| **核心定位** | 通用对话式 AI 助手            | Agentic（代理式）编码工具              |
| **主要场景** | 思考、写作、总结、快速问答、生成内容     | 实际代码开发：改代码、修 bug、重构、自动化任务     |
| **代码访问** | 需手动复制粘贴或上传单个文件         | 自动理解整个代码仓库或文件                 |
| **行动能力** | 只能生成文本，无法直接修改本地文件      | 可读写文件、执行命令、跑测试、Git 操作         |
| **自主性**  | 被动对话式，一步步回应            | 主动 Agent 循环：自主规划、执行、验证        |
| **执行环境** | 浏览器沙盒（受限）              | CLI（本地）、Web 版（云端）、IDE、Desktop |
| **并行任务** | 一般不支持                  | 支持多个任务并行（Web 版尤其强）            |
| **输出结果** | 聊天记录、Artifacts         | 实际修改后的代码、PR、测试通过的分支           |
| **使用门槛** | 极低（打开浏览器即可）            | CLI 版较高，Web 版较低               |

## 2. 其他 Agent VS claude code

**Codex（OpenAI）**
- **Claude Code** 更适合**深度工程工作**：复杂代码库理解、大型重构、调试、架构设计。它在本地运行，开发者在环（interactive），推理能力和代码质量更高
- **Codex** 更适合**自主委托**：云沙盒执行，适合把任务扔给它后台跑（CI/CD、批量任务）。Token 效率更高，成本更可控，GitHub 集成更好

==CodeX 是好工具，但是API配置和文件逻辑还是 claude code 更胜一筹==

**WorkBuddy（腾讯）**
- **Claude Code** 是**专业编码工具**，专注代码仓库操作（读写、测试、Git）
- **WorkBuddy** 是**全能办公工作台**，更偏向国内用户场景：支持消息平台（微信/企业微信）、文档处理、报表自动化、跨系统连接。它不是纯编码 Agent，而是“从想法到交付”的综合 Agent

==这类 Agent 工具还是用国外头部公司的好一些，WorkBuddy 有不少 bug ，还有个人信息的被监控风险==

**Hermes（爱马仕）**
- **专注于**高性能本地运行**的开源 AI Agent 项目，常被视为 Claude Code 的“本地开源替代品”
- 完全本地化运行，支持 Ollama、LM Studio 等本地 LLM
- 优化了长上下文和工具调用效率，适合隐私要求高或无网络环境
- 轻量、部署简单，适合个人开发者或企业内部部署
- **适合场景**：不想把代码上传云端、喜欢本地 LLM、预算有限的用户

**OpenClaw（小龙虾）**
- **开源分布式多代理框架**，强调灵活性和可扩展性
- 支持多 Agent 协作、分布式任务执行
- 高度可定制，可连接各种工具和外部系统
- 社区驱动，更新快，适合有开发能力的用户深度定制
- Claude Code 开箱即用、稳定性更高、模型能力更强（官方 Claude 优化）
- OpenClaw 更灵活，但需要更多配置和维护，代码质量和开箱体验通常不如 Claude Code
- **适合场景**：喜欢自建 Agent 系统、需要多代理协作、或想完全掌控技术栈的开发者/团队

| 维度         | **Claude Code**       | **Hermes**            | **OpenClaw**        | **Codex**     | **WorkBuddy** |
| ---------- | --------------------- | --------------------- | ------------------- | ------------- | ------------- |
| **核心定位**   | 终端优先深度编码代理            | 高性能开源本地 Agent         | 分布式/多代理开源框架         | 云端自主任务代理      | 办公全流程工作台      |
| **交互方式**   | CLI + Desktop/Web/IDE | CLI / 本地界面            | CLI + Web Dashboard | CLI + 云沙盒     | 对话式（微信/飞书等）   |
| **执行环境**   | 本地为主 + 云端             | **完全本地**              | 本地 + 可分布式           | 云沙盒为主         | 本地/混合         |
| **模型支持**   | 仅 Claude 系列           | **多模型**（本地 LLM + API） | 多模型（高度灵活）           | OpenAI GPT 系列 | 多模型 + 国内模型    |
| **代码能力**   | **顶级**（复杂重构）          | 优秀（本地优化）              | 良好（框架灵活）            | 强（自主执行）       | 中等（办公+简单编码）   |
| **自主性**    | 高（Agent Teams）        | 高（本地长期运行）             | **很高**（分布式多代理）      | 最高（云后台）       | 中（流程自动化）      |
| **隐私/本地化** | 良好（本地运行）              | **最佳**（完全本地）          | **优秀**（可自托管）        | 一般（云端）        | 良好（国内合规）      |
| **适用人群**   | 专业开发者                 | 隐私敏感 / 本地 LLM 用户      | 喜欢自建框架的开发者          | 想委托任务的用户      | 国内办公/非纯开发者    |
| **开源程度**   | 部分（核心闭源）              | **高度开源**              | **完全开源**            | 部分开源          | 闭源（企业级）       |
| **定价**     | $20–$200/mo           | API计费                 | API计费               | $20 起         | 国内订阅          |

# 总结

**claude code** 作为 Agent 的开山鼻祖，是目前最强大、最可靠的 AI 自主工作代理工具

**优点非常显著**：
- **自主执行力强**：能独立完成多步骤复杂任务，规划、执行、迭代、验证一气呵成，减少反复跟进的负担
- **工具使用深度**：能真正操作电脑文件、运行命令、整合外部系统，一个靠谱的助手
- **可靠性高**：在长任务、复杂流程中的稳定性优于多数同类工具，不容易“半途而废”或跑偏
- **控制感好**：既有完全自主模式，也有交互确认模式，你可以根据任务重要程度灵活把控

这个 Agent 工具也具备一定的使用门槛，如果将此工具的使用方式和逻辑掌握，那么对 **AI 协作**的理解就会达到一个不错的理解；通过长期的使用和积累复盘可以总结出一套不错的思维逻辑，后续也能将这种思维逻辑迁移到其他的 Agent 工具上
